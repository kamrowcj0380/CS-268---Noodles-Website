Report

  1. Create a website for a client

    Our client can be anyone interested in eating, cooking, or learning about a variety of noodles in the Eau-Claire area.

  2. Include headers

    The above text is a header type, and the text at the very top of the page is a style of header as well as a header type.

  3. 10 or more pages

    1. Index
    2. Contact Us
    3. About Us
    4. Homemade
    5. Soup Noodles
    6. Fried Noodles
    7. Pasta Noodles
    8. Restaurants
    9. History
    10. Fun Facts

  4. Include Navigation Links

    Our navbar is a set of links to navigate the website. These links lead to a few key pages on our website.

  5. Distinct layouts for large/small screens

    Our website uses media queries to resize, and grids become one column when shrunk to the size of a phone.

  6. Consistent style across pages

    We have a style.css folder that is linked on all pages, and this styles everything but the main content of the page. For the main content, we use the same color scheme and general layout across our pages.

  7. Include images with proper alt descriptions

    We have a header image on most pages, and images on the restaurants page. These images all have a short but accurate description.

  8. Declare a favicon

    We use the same favicon across all pages, as it fits the general website pretty well.

  9. Organize the non-HTML resources

    We use an images folder for all of our images, and a css folder for all of our stylesheets.

  10. Do not use libraries or frameworks

    We designed everything on the page based on concepts from class or our own knowledge.

  11. Each page and stylesheet must validate using html/css checkers

    We ran every page and stylesheet through the listed validation services.

  12. The site could be stored using GitHub

    We did, and it can be found here.

  13. File names should be lowercase and alphanumeric

    Our file names are things such as index.html and style.css, which follow these guidelines.

  14. Align content neatly with clearly defined axes

    Pages have a clear line on either end of the main content where there is a margin, and other styling makes use of grids to clearly define the elements.

  15. Make use of multiple font faces

    It isn't easy to notice due to the similar font, but the very top of the page has a header with a unique font.

  16. Include links to external sites that open in a new window

    We have links on our restaurant and recipe pages that open in new windows.

  17. Code must be readable, using both conventional formatting and meaningful id/class names

    We use proper indentation across pages, and our ids are things like "navbar" and classes are things like "navbar_text".

  18. Include a consistent navbar

    The navbar has a style that is used on various pages, and it appears on every page in the same way with the same style.

  19. Include a header and footer section

    Along with the navbar, the header and footer are both included on this page. The header is the text with the blue background, and the footer is at the bottom with the lighter background than the main color of the page.

  20. Use flexboxes and/or grids

    The navbar at the top of the page was implemented using display:grid; in "style.css".
